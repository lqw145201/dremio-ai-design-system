01Overview & Project Structure

The AI Agent prototype is a single-page React application that renders a resizable 4-panel layout. All styling uses CSS custom properties defined in src/styles/theme.css and mapped to Tailwind via @theme inline.

02Design Tokens — Colors

All colors are defined in src/styles/theme.css. Never hardcode a hex value — always reference the CSS token via Tailwind or inline style.

Core Palette

--foreground #202124 Primary text, titles
--secondary-foreground #505862 Secondary text, table headers, labels
--muted-foreground #B0B7BF Placeholders & disabled ONLY
--muted #EEEFF1 Dividers, muted backgrounds
--background #F6F7F8 Page body background
--card #FFFFFF Cards, panels
--border #D2D6DA Input & card borders
--primary #43B8C9 Primary buttons, active states
--sidebar-primary #2E92A1 Primary button hover
--accent #008489 Links, ghost button text
--destructive #CA3F32 Delete, error states
--destructive-hover #AD3021 Destructive button hover
--background-hover #F1FAFB Ghost button hover bg, nav hover
--sidebar #2A394A Left nav background
--chart-5 / info #277ABD Info callouts, TimeBadge
Critical: --muted-foreground is for placeholders and disabled states ONLY. Table headers, stat labels, and section descriptions must use --secondary-foreground (#505862).
⚠ Prototype inconsistency — primary vs accent as button background.
The formal <Button variant="default"> component correctly uses --primary (#43B8C9) as its background. However, several action buttons in the prototype (Execute, Run in SQL runner, Run in SQL in WorkspacePanel) were built as ad-hoc <button> elements with bg-accent (#008489) — a darker teal — applied directly.

Rule: --accent is for ghost button text and links only — never as a button background fill. Any hand-built action button using bg-accent should be refactored to use <Button variant="default"> (which renders with --primary).

13Panel Layout & Interactions

Resize Behavior

  • All panels use react-resizable-panels — PanelGroup + Panel + PanelResizeHandle
  • Resize handles are 4px wide, invisible by default, show a 1px --border line on hover
  • Panels have minSize prop to prevent collapse below usable size
  • State persisted in usePanelLayout hook (localStorage)

Collapse / Expand

  • Each side panel has a collapse button (chevron icon, 24px) in its header
  • Collapsed state: panel unmounts, CollapsedPanelStrip renders in its place
  • Re-expand: click strip or chevron to restore last saved width

Portal-Rendered Dropdowns

  • Dropdowns that overflow panel bounds use ReactDOM.createPortal into document.body
  • Position calculated with getBoundingClientRect() on the trigger element
  • Close on: outside click, Escape key, scroll of the trigger's scroll container

14Critical Rules

These are the most common mistakes. Read before making any change.

  1. --muted-foreground for placeholders/disabled ONLY. Table headers, stat labels, section descriptions → --secondary-foreground.
  2. No card chrome on full-page layouts. Pages are h-screen w-screen. Never add border/radius/shadow/margin to the root page container.
  3. Max 1 Primary button per page. All other actions use Secondary or Ghost.
  4. Row actions are hover-only. Use opacity-0 group-hover:opacity-100. Never show them unconditionally.
  5. Delete icons are neutral by default. Apply text-destructive on hover only — never always-red.
  6. Secondary nav right border uses --muted, not --border. border-right: 1px solid var(--muted).
  7. Icon sizes are not interchangeable. 24px (nav/header), 20px (table rows), 16px (button inline).
  8. cva variant hover cannot be overridden via className. Tailwind resolves by stylesheet order. Edit the variant in button.tsx directly.
  9. Always read Figma before building any component. Call get_design_context first. Never approximate from memory.
  10. All icons are in src/app/components/icons/. Check there before installing any icon package or using Fluent UI.
  11. Read the page node first, then each child. Never start coding until the full Figma hierarchy is mapped.
  12. Never place UI elements by intuition. Run/Preview/Engine controls belong in the Action Header, not the Top Nav.
  13. Never guess icon names. Read the component's children in Figma to see exactly which icon slot is used.
  14. Never use UI design tokens for code syntax colors. --primary, --accent, --destructive are for UI chrome only. SQL editors use a code-editor palette (keywords #0033B3, strings #067D17, functions #7A3E9D).
  15. Component name is the source of truth. If it's named Nav/Top Nav, read what's actually inside — don't assume based on the word "nav."
