01Overview & Project Structure

The AI Agent prototype is a single-page React application that renders a resizable 4-panel layout. All styling uses CSS custom properties defined in src/styles/theme.css and mapped to Tailwind via @theme inline.

Portable Icon Strategy

This design system is fully portable — it contains no generated svgPaths imports. All icon SVG data is resolved using one of three strategies:

StrategyWhen to useExample
Named icon component (IconXxx)Standard Dremio icons matching a file in components/icons/<IconSearch size={20} />
Inline PATH_XXX constantNon-standard viewBox icons unique to a specific component (not generic icons)const PATH_CHEVRON = "M9.69…"
Standard CSS animationSpinners / loading statesborder-t-transparent animate-spin
Inline PATH constants are declared at the top of the component file. The viewBox values must exactly match the path data — they are specific to each icon and are not interchangeable with standard 24×24 icon sizes.

Token clarification — primary vs accent.
--accent (#008489) is the primary button background color. --primary (#43B8C9) is the lighter teal used for active state tints, running spinners, and highlights — not for button backgrounds. Ad-hoc action buttons (Execute, Run in SQL runner) using bg-accent are correct — they should be refactored to use <Button variant="default"> for consistency, not re-colored.

09cToast Notification

Source: components/Toast.tsx. A lightweight notification that slides in from the bottom-right corner and auto-dismisses after 3 seconds. Used for copy confirmations, save feedback, errors, and other transient status messages.

Why bottom-right? The AI agent layout has a chat input at the bottom of the left panel and workspace content filling the center/right. Bottom-right is the neutral corner furthest from both active regions — it matches the convention of developer tools (VS Code, Cursor, Linear, Vercel) and avoids cutting across content being read.

13Panel Layout & Interactions

Resize Behavior

  • All panels use react-resizable-panels — PanelGroup + Panel + PanelResizeHandle
  • Resize handles are 4px wide, invisible by default, show a 1px --border line on hover
  • Panels have minSize prop to prevent collapse below usable size
  • State persisted in usePanelLayout hook (localStorage)

Collapse / Expand

  • Each side panel has a collapse button (chevron icon, 24px) in its header
  • Collapsed state: panel unmounts, CollapsedPanelStrip renders in its place
  • Re-expand: click strip or chevron to restore last saved width

Portal-Rendered Dropdowns

  • Dropdowns that overflow panel bounds use ReactDOM.createPortal into document.body
  • Position calculated with getBoundingClientRect() on the trigger element
  • Close on: outside click, Escape key, scroll of the trigger's scroll container

14Critical Rules

These are the most common mistakes. Read before making any change.

  1. --muted-foreground for placeholders/disabled ONLY. Table headers, stat labels, section descriptions → --secondary-foreground.
  2. No card chrome on full-page layouts. Pages are h-screen w-screen. Never add border/radius/shadow/margin to the root page container.
  3. Max 1 Primary button per page. All other actions use Secondary or Ghost.
  4. Row actions are hover-only. Use opacity-0 group-hover:opacity-100. Never show them unconditionally.
  5. Delete icons are neutral by default. Apply text-destructive on hover only — never always-red.
  6. Secondary nav right border uses --muted, not --border. border-right: 1px solid var(--muted).
  7. Icon sizes are not interchangeable. 24px (nav/header), 20px (table rows), 16px (button inline).
  8. cva variant hover cannot be overridden via className. Tailwind resolves by stylesheet order. Edit the variant in button.tsx directly.
  9. All icons are in components/icons/. Check there before installing any icon package or using Fluent UI.
  10. Never place UI elements by intuition. Each component's name defines exactly what belongs inside it — verify before adding anything.
  11. Never use UI design tokens for code syntax colors. --primary, --accent, --destructive are for UI chrome only. SQL editors use a code-editor palette (keywords #0033B3, strings #067D17, functions #7A3E9D).
  12. Never import svgPaths from generated files. This design system is portable — all icons use named IconXxx components or inline PATH_XXX constants. Never add import svgPaths from "../../imports/svg-*".
  13. AI responses have no message bubble. Only user messages get UserMessageBubble. AI text is plain, unboxed, left-aligned. Never wrap AI output in a card or tinted background.
  14. UserMessageBubble uses asymmetric radius. All corners 8px except bottom-right which is 4px — the "speech-bubble tail" side. Never apply uniform rounded-lg to user messages.
